# Memory Priority Rules

**Created:** 2026-04-02  
**Rule:** Local First, GitHub as Backup

---

## 🎯 读取优先级（高 → 低）

```
1. 本地 memory/*.md 文件（最高优先级）
2. GitHub 备份（检查更新）
3. 飞书文档（协作补充）
```

---

## 📥 会话开始时的流程

### 第一步：读本地（优先）
```bash
# 立即读取，不依赖网络
ls ~/.openclaw/workspace/memory/*.md
```

### 第二步：检查 GitHub（可选）
```bash
# 网络好时：git pull 获取更新
# 网络差时：跳过，使用本地版本
```

### 第三步：合并策略
```
if 本地文件比 GitHub 新:
    使用本地版本
elif GitHub 有新内容:
    合并到本地
else:
    使用本地版本
```

---

## 📤 会话结束时的流程

### 第一步：写本地（必须）
```bash
# 立即写入，不依赖网络
echo "更新内容" >> memory/2026-04-02.md
```

### 第二步：推 GitHub（异步）
```bash
# 后台推送，失败不重试（避免阻塞）
git push origin main &
```

---

## ⚡ 为什么本地优先？

| 场景 | 本地优先优势 |
|------|-------------|
| **网络中断** | 完全不受影响 |
| **响应速度** | 毫秒级读取 |
| **数据安全** | 本地是真相源 |
| **离线工作** | 无需网络也能记住 |

---

## 🔄 冲突解决

### 情况 1: 本地新，GitHub 旧
```
动作：推送本地到 GitHub
结果：GitHub 被覆盖
```

### 情况 2: GitHub 新，本地旧
```
动作：拉取 GitHub 到本地
结果：本地被更新
```

### 情况 3: 都有新内容（冲突）
```
动作：手动合并
结果：保留两者信息
```

---

## ✅ 当前实现

**读取**: 本地 `memory/*.md`  → 即时响应 ✅  
**写入**: 本地 → GitHub      → 异步备份 ✅  
**飞书**: 读取 → 补充协作信息 → 可选 ✅

**本地永远是真相源！**
