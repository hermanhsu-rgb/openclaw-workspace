## 推送问题修复

### 问题
- 推送消息显示为乱码，不在对话中正常显示

### 根因
- `delivery.mode: "announce"` 会发送到公告区域而非对话
- 需要设置 `mode: "direct"`

### 重要发现
- `openclaw cron add` 默认设置 `mode: "announce"`
- `openclaw cron edit` 会重写delivery字段，丢失mode
- 每次用openclaw命令操作后需要手动改回direct

## 最终解决方案（12:00更新）

### 关键认知转变
用户指出：不要跟系统对着干，用官方规则！

**问题根源**：
- 手动修改jobs.json设置`mode:direct`会被Gateway覆盖
- 一直在跟系统强碰，这是错的

**正确做法**：
- 使用官方支持的`--announce`模式
- 不再手动修改jobs.json
- 让系统按自己的规则运行

### 重建所有cron任务（12:00）

删除所有旧任务（保留Memory Dreaming Promotion），用官方规则重建：

```bash
openclaw cron add \
  --name "任务名" \
  --cron "表达式" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --wake now \
  --channel wecom \
  --to "heng-kaihsu" \
  --message "任务内容" \
  --announce  # 官方规则！
```
