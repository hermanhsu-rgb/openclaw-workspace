## 1. WeCom发图收图正确方式

### 用户侧（发送）
- 企业微信中**一次性选择多张照片**发送（≤500ms内）
- 系统自动防抖合并（500ms窗口）
- 建议：一次选完所有照片后点击发送，不要分开发

### AI侧（接收）
1. WebSocket收到多条ws-inbound消息（每张图一条）
2. 防抖机制将500ms内到达的消息合并成一个批次
3. `mergedCount=N`表示合并了N张图
4. 所有图片保存到`~/.openclaw/media/inbound/`目录
5. AI可以同时看到所有图片并进行分析

### 验证结果
- ✅ 3张图：mergedCount=3，保存3个文件
- ✅ 4张图：mergedCount=4，保存4个文件
- ✅ 5张图：分2批（4+1），共保存5个文件

### 注意事项
- 发送间隔超500ms会被分到不同批次（正常行为）
- 文件位置：`~/.openclaw/media/inbound/`
- 修复文件：`monitor/state.ts`（去`!activeIsInitial`）、`monitor.ts`（用saveMediaBuffer API、循环从i=0）
