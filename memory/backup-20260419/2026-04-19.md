

## 2026-04-19 16:00 - WeCom发图正确收图流程（已修复）

### 问题背景
用户从企业微信发送多张照片时，AI原来只收到1张。

### 修复后的正确流程

**用户侧（发送）**：
- 在企业微信中一次性选择多张照片发送（≤500ms内发完）
- 系统会自动防抖合并（500ms窗口）
- 建议：一次选择所有照片后点击发送，不要分开发

**AI侧（接收）**：
1. WebSocket收到多条ws-inbound消息（每张图一条）
2. 防抖机制将500ms内到达的消息合并成一个批次
3. `mergedCount=N`表示合并了N张图
4. 所有图片保存到`~/.openclaw/media/inbound/`目录
5. AI可以同时看到所有图片并进行分析

### 验证结果
- ✅ 3张图：mergedCount=3，保存3个文件
- ✅ 4张图：mergedCount=4，保存4个文件
- ✅ 5张图：分2批（前4张+后1张+文字），共保存5个文件

### 技术细节
- 防抖窗口：500ms（DEFAULT_DEBOUNCE_MS）
- 批次合并：允许初始批次合并（移除了`!activeIsInitial`限制）
- 媒体保存：使用`core.channel.media.saveMediaBuffer()` API
- 文件位置：`~/.openclaw/media/inbound/`

### 注意事项
- 如果发送间隔超过500ms，会被分到不同批次
- 这是正确行为，确保每批消息得到适当的响应
- 每批独立处理，不会漏掉任何图片

### 相关文件
- `/root/.openclaw/extensions/wecom/src/monitor/state.ts` — 防抖合并逻辑
- `/root/.openclaw/extensions/wecom/src/monitor.ts` — 媒体保存逻辑
