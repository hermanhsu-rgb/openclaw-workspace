# 跨平台记忆同步问题记录

**Status:** 🔴 未解决  
**发现时间:** 2026-04-02  
**严重程度:** 高

---

## ❌ 问题描述

**不同平台使用不同存储机制：**

| 平台 | 存储位置 | 格式 | 状态 |
|------|----------|------|------|
| **企业微信** | `memory/*.md` | Markdown | ✅ 可读 |
| **飞书** | `memory/main.sqlite` | SQLite | ❌ 未同步 |
| **Telegram** | `memory/main.sqlite` | SQLite | ❌ 未同步 |

**结果**: 企业微信无法看到飞书/telegram 的对话记录

---

## 🔍 测试失败

**测试**: 在飞书告诉密码 → 在企业微信询问
**结果**: ❌ 不知道密码
**原因**: 企业微信只读取 .md 文件，飞书写入 .sqlite

---

## 🔧 解决方案

### 方案 1: 统一写入 Markdown (推荐)

**修改飞书/telegram 渠道**: 会话结束时同时写入
- `memory/main.sqlite` (原有)
- `memory/*.md` (新增)

**优点**: 
- 所有平台可读 Markdown
- 人类可读的备份
- 易于版本控制

### 方案 2: 统一读取 SQLite

**修改企业微信**: 会话开始时同时读取
- `memory/*.md` (原有)
- `memory/main.sqlite` (新增)

**优点**:
- 保持原有架构
- SQLite 查询灵活

### 方案 3: 同步脚本 (临时)

**创建定时脚本**: `sync-memory.sh`
- 定期从 SQLite 导出到 Markdown
- 合并所有平台记忆

**缺点**:
- 非实时
- 需要手动/定时触发
